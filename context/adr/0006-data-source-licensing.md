# ADR-0006: データソースのライセンス表記とデュアルライセンス構成

## ステータス
accepted

## 日付
2026-06-22

## コンテキスト
- ユーザ要求: 「データソースのライセンス表示って必要かどうか調べて」→ 調査の結果「必要」と判明し、「この3点を実装して」と指示。
- 中核データソース **KanjiDic2**（`scripts/generate_data.py` が取得）は EDRDG（電子辞書研究開発グループ）の所有物で、ライセンスは **Creative Commons Attribution-ShareAlike 4.0 International（CC BY-SA 4.0）**（[EDRDG licence](https://www.edrdg.org/edrdg/licence.html) で確認）。
- CC BY-SA 4.0 は2つの義務を課す:
  1. **BY（表示）**: 出典の謝辞。EDRDG はアプリ/サイトについて「各画面表示で、またはスマホアプリ等では『About』のような別画面で謝辞を出すこと」を明示要求。
  2. **SA（継承）**: 派生物は同一/互換ライセンスで配布。EDRDG の著作権を主張してはならない。
- 当時の状況: アプリ画面（右下 ⓘ info ボード）に出典表記なし、リポジトリに LICENSE なし。`public/data/kanji-3d.json` は KanjiDic2 の意味(`m`)・読み(`on`/`kun`)をそのまま含む **派生物** であり SA の対象。
- 他のデータ生成要素（Embedding モデル all-mpnet-base-v2 = Apache-2.0、UMAP/scikit-learn = BSD）はビルド時のみ使用し再配布しないため、生成物への表示義務はなし。

## 検討した選択肢

### 選択肢A: 全体を CC BY-SA 4.0 に統一
- メリット: 単一ライセンスでシンプル。SA の整合が取りやすい。
- デメリット: コードの再利用にも SA（継承）義務が付き、アプリコードの再利用性が下がる。コードはデータの派生物ではないのに不要な制約を被る。

### 選択肢B: デュアルライセンス（コード = Apache-2.0 / データ = CC BY-SA 4.0）
- メリット: コードは寛容ライセンスで再利用しやすく、データ部分のみ SA を継承。CC のコピーレフトは「翻案物」に及ぶもので、データを表示するコードは翻案物ではないため分離が法的に妥当。
- デメリット: ライセンスが2本になり、どこに何が適用されるかを明示する手間が要る。

## 決定
選択肢B（デュアルライセンス）を採用。コード本体は **Apache-2.0**（ユーザ選択）、`public/data/` 配下の生成データは **CC BY-SA 4.0**。

実装:
1. アプリの info ボード（About 画面に相当）に KanjiDic2 の出典謝辞を常設（`index.html`）。EDRDG の画面表示要件を満たす。
2. ルート `LICENSE`（Apache-2.0 正文）/ `public/data/LICENSE`（CC BY-SA 4.0 + KanjiDic2 謝辞、データと同梱で SA を満たす）。
3. `README.md` / `README.ja.md` に License 節を追加し2本立てを明記。

## 理由
- コードとデータでライセンス性質が異なる（データのみ KanjiDic2 由来で SA 継承）ため、両者を分けるのが法的に正確かつコードの再利用性を損なわない。
- Apache-2.0 は寛容かつ特許グラント・貢献者保護があり、データの CC BY-SA 4.0 と共存可能（CC のコピーレフトは翻案物にのみ及び、別個の著作物であるコードには波及しない）。
- `public/data/LICENSE` をデータと同ディレクトリに置くことで、データ単体が再配布されても SA の「同ライセンス + 出典」が物理的に随伴する。

## 影響
- データ（`public/data/`）を再配布・改変する場合は CC BY-SA 4.0 を継承し、KanjiDic2/EDRDG への出典表示を保持する必要がある。コードは Apache-2.0 で自由に再利用可。
- 今後 KanjiDic2 以外のデータソースを追加する場合、そのライセンスの BY/SA 互換性を都度確認すること。
- `LICENSE` の著作権表記は暫定で「Kanji-Verse contributors」。正式な権利者名があれば差し替える。
