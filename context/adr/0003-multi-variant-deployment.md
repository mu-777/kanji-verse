# ADR-0003: マルチバリアント並列デプロイ構成

## ステータス
accepted

## 日付
2026-03-16

## コンテキスト
- 「宇宙の壮大感を出したい」というユーザー要求
- 複数のビジュアライゼーションアプローチを実装して比較したい
- ユーザー発言: 「正直、実装して見てみないと判断がつかない」

## 検討した選択肢

### 選択肢A: 単一ページでバリアントを切り替え
- メリット: URLが1つ、共通コードが多い
- デメリット: バリアントごとに大きく異なるコードの混在が複雑

### 選択肢B: Vite マルチページアプリ（採用）
- メリット: 各バリアントが完全独立、並行して閲覧・比較できる、GitHub Pages で全バリアントが同時デプロイ
- デメリット: HTML ファイルが増える（CSSの重複）

## 決定
選択肢B（Vite マルチページ）を採用。

## 理由
- バリアント間でレンダラーのアーキテクチャが根本的に異なる（Canvas 2D vs Three.js）ため、切り替え型よりも独立ページの方がシンプル
- 並行ブラウズ（タブを並べて比較）が自然にできる
- Vite のマルチページサポートが成熟しており、設定コストが低い

## 影響
- `base: "/kanji-verse/"` を含む vite.config.ts でビルド済み
- GitHub Actions の deploy.yml は変更不要（dist/ をそのまま配信）
- 共通 UI ロジックは `src/shared/ui.ts` の `UIRenderer` インターフェースで抽象化
- ネビュラ差し替えは `NebulaEngine` インターフェース（src/three-nebula/nebula-engine.ts）経由
