# ADR-0005: Google アナリティクス導入とプライバシー開示方針

## ステータス
accepted

## 日付
2026-06-21

## コンテキスト
- ユーザー要求（引用）: 「Googleアナリティクスを設定したい。gtagは発行済（G-E6F07KS3MG）。これを適切な場所に置いといて。あと他にGoogleアナリティクスを使うのにケアすべきことは？」
- 公開サイト（GitHub Pages）でアクセス解析を行うにあたり、Google アナリティクス利用規約は GA と Cookie の使用を利用者に**開示すること**を要求する。
- 本サイトは星空風の世界観を持つ単一ページアプリ（ADR-0004）。開示を素のテキストリンクで足すと世界観が崩れる、というのがユーザーの懸念。
- 既存に welcome オーバーレイ（Inter ExtraLight / 横罫 / グローの「ボード」意匠）があり、これを流用できる。
- ユーザー判断で確定した前提: localhost ガードは不要 / 開示は「開示文＋シンプルなポリシー」で十分 / カスタムイベント計装は一旦不要 / 文言は英語のみ。

## 検討した選択肢

### 選択肢A: フッターに素のテキストリンクで Privacy ページへ
- メリット: 実装が最小。`public/privacy.html` を置くだけ。
- デメリット: 星空の世界観に対して異物感がある。常時表示のテキストが景観を損なう。

### 選択肢B: 右下に ⓘ アイコン → welcome と同じボード調のポップアップで集約
- メリット: 既存意匠を再利用し世界観を維持。About・操作ヒント・アクセス解析の開示を1枚に集約できる。常時表示だった `#zoom-hint`（操作ヒント）も「info」としてここへ統合でき、メインビューがすっきりする。
- デメリット: マークアップ／JS が少し増える。操作ヒントが常時表示でなくなる。

## 決定
選択肢Bを採用する。
- gtag.js スニペットを `index.html` の `<head>`（charset は先頭維持、async）に配置。
- 右下に ⓘ アイコン（`#info-btn`）。クリックで `#info` ボードを開閉（再表示可能、表示制御は `main.ts`）。
- info ボードの内容: About（アプリ説明）＋操作ヒント（Drag/Scroll/Click）＋アクセス解析(GA)の開示文＋ opt-out / Google データ利用ページへのリンク。**英語のみ**。
- 常時表示の `#zoom-hint` は廃止し、操作ヒントは info ボードと welcome（初回）に集約。
- welcome の CSS を `.kv-*` 共通クラス（`.kv-overlay` / `.kv-board` / `.kv-title` / `.kv-desc` / `.kv-hint` / `.kv-btn`）に切り出し、welcome と info で共有（DRY）。
- 同意バナー（Consent Mode v2）は導入しない。

## 理由
- 世界観の一貫性とコストを両立できるのがBのみ。素リンク（A）は意匠を壊す。
- zoom-hint と開示を info に統合することで、常時表示の UI を1つ減らしつつ情報量はむしろ増やせる。
- 個人プロジェクトであり EU 圏への本格対応はスコープ外。日本の慣行上、GA のみなら「開示＋簡易ポリシー」で実務上十分との判断。

## 影響
- 操作ヒントが常時表示でなくなる。初回 discoverability を担保するため welcome の hint に操作行（Drag/Scroll/Click）を追記済み。
- **localhost（dev: 5173 / preview: 4173）のテストアクセスも GA に計上される**（ガード不要の判断）。GA の数値は実数でなく傾向として扱う。広告ブロッカーによる欠測も同様。
- 計測 ID `G-E6F07KS3MG` はクライアントに露出するが、これは仕様上の公開情報で秘匿不要。
- IP は GA4 でデフォルト匿名化されるため `anonymize_ip` 指定は不要。
- 将来 Content-Security-Policy を導入する場合、`googletagmanager.com` / `google-analytics.com` の許可が必要。
