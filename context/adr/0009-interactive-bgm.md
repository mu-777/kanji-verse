# ADR-0009: インタラクティブBGM（Web Audio 単一ベッド＋adaptive DSP / AI生成音源 / ミュート既定）

## ステータス
accepted

## 日付
2026-06-24

## コンテキスト
- ユーザの要求（引用）: 「BGMをつけたい。できればインタラクティブに変化するようなもの。シーンを操作しているとき、していないときで緩急をわけるなど。」
- ヒアリングで固めた方向性:
  - **ムード**: 温かく漂う“宇宙の憧れ”系メロディック・アンビエント。**和なし／畏怖なし／冷たさなし／ガラス感なし**。常時BGM感が強く、Carbon Based Lifeforms「Photosynthesis」的な柔らかい脈動を持つ（試聴で An Ending / Cornfield Chase / Photosynthesis / Abandon Window を「良い」、1/1 / Dream 3「静止しすぎ」、Sea Wall「畏怖」/ Ikeda「冷たすぎ」を却下）。
  - **音源**: Google **Gemini** で生成した **静かな脈動ベッド1本**（`public/audio/bgm.mp3`, 192kbps / 約2:48 / 3.9MB）。ユーザーよりライセンス上の問題なしと確認。
  - **緩急の強さ**: 中間（基本は一定の常時BGM、操作で「少し明るく・前に出る」がそれとわかる程度）。
- 制約:
  - **オートプレイ規制**: ユーザージェスチャまで音は出せない。
  - **起動が既に遅い**（~16s、テクスチャ生成 ~14s）。音は遅延ロードで起動をブロックしない。
  - 軽量方針（依存・バンドルを増やしたくない）。
  - モバイル/iOS Safari の AudioContext resume、タブ非表示のダッキング。

## 検討した選択肢

### 選択肢A: Tone.js でフル生成ハイブリッド（当初案）
- メリット: 生成アルペジオ/ベル・無限変化・データ連動（クラスタ→音）。
- デメリット: スコープが**1本ベッド＋ガラスなし**に固まり生成要素がほぼ消えた。ライブラリ重量の割に使わず、起動/バンドルに不利。

### 選択肢B: 素の Web Audio API ＋ 単一ベッド ＋ adaptive DSP
- 構成: ベッドをループ再生し、`energy`（操作の指数減衰スカラー）で **lowpass cutoff ＋ 動的ゲイン**を駆動。「薄いレイヤー追加」感は広いフィルタスイープで表現。
- メリット: **依存ゼロ・軽量・起動非ブロック**。中間の緩急に十分。プロジェクトの軽量方針と整合。
- デメリット: 1本ループゆえ長時間で飽きの可能性（→ 緩急＋フィルタ変化で緩和。足りなければベッド追加）。

### 選択肢C: 既成ステム多層レイヤリング
- メリット: 音楽的クオリティ最高。
- デメリット: 音源1本のみの現状と不一致。同期stemの調達コスト。

## 決定
**選択肢B（素の Web Audio API）を採用する。** ただし「ベッド＋DSP のみ」から、反復チューニングを経て
**ベッド＋2つの生成インタラクション層**へ発展（生成は素の Oscillator で実現し、Tone.js は不要のまま）。

最終実装（すべて `src/three-core/audio.ts` に隔離）:
- **ベッド**: `BufferSource(loop)` → `lowpass` → `dynGain` → `master`。`energy`（操作の指数減衰スカラー＝検索/ダイブ/運動）で `cutoff`(≈620→8200Hz)・`dynGain`(0.78→1.0) を駆動。
- **ドラッグ層(space)**: `camera.getSpeed()`→`motion` 連動。浮遊パッド（基音＋5度＋オクターブ＋上倍音）＋薄い空気ノイズ＋ゆっくり LFO →（motion で）lowpass を開きつつゲイン。**非対称エンベロープ**（attack 0.06s／release 1.4s）で急停止でも余韻を残す。
- **クリック層(glass)**: 非整数倍音のベル＋**合成IRリバーブ**（`ConvolverNode`、素材不要）の余韻。ピッチは**視点（カメラ）から漢字までの距離**を音域バイアスにした**ペンタトニック旋律ウォーク**で決定（連続同音を禁止＝単調回避、クリックで小メロディが紡がれる）。距離計算はクリック時1回のみ。
- 既定ミュート。ON にした時点で初めて遅延ロード。`AudioContext` はジェスチャ内 `resume`、ブロック時は次ジェスチャで復帰。`localStorage(kv_sound)`、`visibilitychange` でダッキング＋suspend。
- UI: 右下 `#sound-btn`（off で取り消し線）＋ welcome の `#welcome-sound`（同期）。demo は自動 ON、`cinematic.ts` で `#sound-btn` 非表示。
- 全層が `master` を経由 → 動画シェア録画（ADR-0010）にそのまま乗る。

音色決定のプロセス: 一時的に `?audiolab=1` の聴き比べパネル（複数の drag/click 音色を実機で切替）を設けてユーザーが耳で選定 →
**Drag=space / Click=glass（旋律ウォーク）** に確定後、パネルと未使用音色・選択APIは撤去した。

## 理由
必要な生成要素（パッド合成・ベル・旋律ウォーク・リバーブ）は素の Oscillator/Convolver で実現でき、**Tone.js は終始不要**＝依存ゼロでバンドル/起動に優しい（既存の起動遅延問題に整合）。
「緩急が間接的すぎる」というFBに対し、ベッドのDSPだけでなく**操作に直接連動する生成層**（運動→風・クリック→音）を足したことで反応が明確化。
さらに「毎回同じ音は単調・純ランダムは散漫」というFBに対し、**データ（視点距離）バイアス付きの旋律ウォーク**で「音楽的かつ非反復かつデータ連動」を同時に満たした。
オートプレイ規制・起動非ブロック・ミュート既定で、再訪ユーザーや音を望まない閲覧者にも礼を失しない。

## 影響
- 音色を変えたいときは `audio.ts` 冒頭の定数（`DRAG_*`・`MOTION_*`・`PENTA`・`IDLE/ACTIVE_CUT`・`BASE_VOLUME`）で調整。聴き比べが要るなら `?audiolab=1` のパネルを git 履歴から復活させられる。
- 1本ループの飽きが課題化したら、ベッド追加（クロスフェード）で対処。
- BGM は Gemini 生成（provenance を本ADR/PROJECT.md に記録）。新たな音源を足す際はライセンスを都度確認。
- `public/audio/bgm.mp3`（3.9MB）はビルド成果物に含まれるが、遅延ロードのため初期表示はブロックしない。
