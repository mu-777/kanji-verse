# 計画: favicon の作成

## 日付
2026-06-21

## 目的
ユーザ発言: 「適切なfaviconを作って」。
方向性のヒアリングに対しユーザ提案: 「Kanji-Verseの訳の漢・字・宇・宙を組み合わせるようなのはどう？ faviconで字が読める必要はないので重なるように組み合わせてスペースを小さくするとか」。

→ 「Kanji-Verse」の語源 **漢字**（kanji）＋ **宇宙**（uchū=verse/cosmos）の4文字を、
深宇宙の背景に発光する星のクラスタとして1タイルに凝縮したブランドマークを作る。
アプリ本体（夜空に漢字が星として浮かぶ）の世界観を favicon に落とし込む。

## 現状把握
- これまで favicon は未設定（index.html に link なし、public に画像なし）。
- ブランド配色: 背景 #05050f、星/文字 #e0e4ff〜rgba(120,140,255)、人名アクセント金 #ffd98e。
- 既存の OGP（public/ogp.png）は密な漢字星空＋Inter ExtraLight ロゴ。faviconはこの世界観を踏襲。
- 環境: WSL2。CJK フォントはシステム未導入だが Windows 同梱フォントが /mnt/c/Windows/Fonts で利用可能。
- 既存ツール: @napi-rs/canvas（generate-ogp.mts で使用）。

## アプローチの選択肢
### A: 1文字の発光漢字（字／永／名 など）
- メリット: 16px でも判別しやすい。シンプル。
- デメリット: 「Kanji-Verse」という固有名のニュアンスは出ない。
### B: 漢・字・宇・宙の4文字を重ねた発光クラスタ（採用）
- メリット: 語源そのもの。アプリ本体（漢字の星空）の凝縮。固有性が高い。
- デメリット: 16px では4つの複雑な漢字が潰れる（ユーザーは読めなくてよいと合意済み）。
### C: 文字なしの抽象的な星団
- メリット: 言語非依存。小サイズに強い。
- デメリット: 「漢字アプリ」と分からない。

## 決定
**選択肢B** を採用。グリフのアウトライン（ベクターパス）を単一の真実とし、SVG とラスタ（PNG/ICO）を同一配置から生成する。
- フォント: 游明朝 Regular（Windows 同梱 `yumin.ttf`）。明朝体の強弱が Inter ExtraLight の世界観と調和。
  opentype.js は `.ttc`（游ゴシック等）非対応のため、plain TTF の游明朝を採用。
- 配置: 2x2（読み順 漢字／宇宙 を保持）、適度な重なり、明暗をばらつかせ 宙 だけ金でハイライト。

## 検証方法
- 512px / 180px / 32px / 16px を実画像で目視確認（明背景・暗背景）。
- `npm run build`（CI と同一の tsc && vite build）が通り、dist ルートに4アセットが出力され、
  dist/index.html に link が残ることを確認。
- base `/kanji-verse/` 配下で相対パスが解決することを確認（相対パス href を使用）。
